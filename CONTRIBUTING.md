# Guía de Contribución - SaaSImperialBarber

Para cumplir con los criterios de evaluación de Desarrollo Web Integral, seguimos estrictamente el flujo de trabajo de **GitFlow**.

## 📌 Reglas de Oro
1. Está totalmente prohibido hacer commits directos a las ramas `main` o `develop`.
2. Todo cambio en el código debe nacer de un Issue previamente asignado en el tablero de GitHub.
3. Para trabajar en una tarea, crea una rama local con la estructura: `feature/nombre-de-tu-tarea`.
4. Al finalizar, abre un Pull Request (PR) apuntando a la rama `develop` y etiqueta al Tech Lead para su revisión y aprobación.

## 👥 Cuotas Técnicas de la Semana
* Mínimo 3 commits individuales por integrante.
* Vinculación obligatoria de Pull Requests con su respectivo Issue.
